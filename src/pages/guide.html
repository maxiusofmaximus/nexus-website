Bienvenido a NexusEngine

NexusEngine es una plataforma revolucionaria que combina múltiples motores gráficos con inteligencia artificial para crear una experiencia de desarrollo de juegos única y accesible.

Múltiples Motores

Kaplay, motor personalizado offline, y próximamente Phaser y PixiJS.

IA Integrada

Asistente inteligente con múltiples proveedores y modos de interacción.

Comandos Naturales

Programa usando palabras clave humanizadas en español.

Funciona Offline

Motor personalizado que no requiere conexión a internet.

¿Nuevo en NexusEngine? Te recomendamos empezar con la sección de "Métodos de Interacción" para entender las diferentes formas de usar la plataforma.

Motor Personalizado (Offline)

El motor personalizado está diseñado para funcionar completamente offline con una API simplificada.

Creación de Objetos
// Crear sprite
engine.createSprite({
    x: 100, y: 200,
    width: 32, height: 32,
    color: '#FF0000'
});

// Crear rectángulo
engine.createRect({
    x: 200, y: 400,
    width: 200, height: 20,
    color: '#00FF00'
});
Controles
// Eventos de teclado
engine.input.onKeyPress('Space', () => {
    player.jump(500);
});

// Mouse
engine.input.onMouseClick((x, y) => {
    console.log(`Click en: ${x}, ${y}`);
});
Ventaja Offline: Este motor funciona sin conexión a internet, ideal para desarrollo en entornos sin conectividad.

Modos de Inteligencia Artificial

Modo Ask

Solo proporciona información y responde preguntas. No ejecuta código.

Ejemplos:
  • "¿Cómo creo un personaje?"
  • "¿Qué comandos hay disponibles?"
  • "Explícame la física en Kaplay"
Solo consultas
Modo Agent

Puede ejecutar comandos y generar código automáticamente.

Ejemplos:
  • "Crea un personaje que salte"
  • "Agrega un enemigo que se mueva"
  • "Implementa un sistema de puntuación"
Ejecuta acciones
Modo Híbrido

Decide automáticamente entre Ask y Agent según el contexto.

Ventajas:
  • Detección inteligente
  • Respuesta apropiada
  • Experiencia fluida
Inteligente
Consejo: Usa el modo Ask para aprender y el modo Agent para crear. El modo Híbrido es ideal para usuarios que quieren la mejor experiencia automática.

Ejemplos Prácticos

Juego de Plataformas
// Configurar juego
inicializar juego 800 600

// Crear personaje
crear jugador 100 300
aplicar gravedad jugador
aplicar física jugador

// Crear plataforma
crear plataforma 0 550 800 50

// Controles
al presionar izquierda jugador mover -200 0
al presionar derecha jugador mover 200 0
al presionar espacio jugador saltar 500

// Enemigo
crear enemigo 400 300
aplicar gravedad enemigo
Sistema de Recolección
// Variables
crear variable puntuacion 0
crear variable monedas_recolectadas 0

// Crear monedas
crear moneda 200 300
crear moneda 400 250
crear moneda 600 350

// Colisión con monedas
al tocar jugador moneda:
    destruir moneda
    sumar puntuacion 100
    sumar monedas_recolectadas 1
    reproducir sonido "coin"

// Mostrar puntuación
crear texto puntuacion 20 20
Efectos Visuales
// Partículas de explosión
al tocar bala enemigo:
    crear particulas explosion enemigo.x enemigo.y
    destruir enemigo
    destruir bala
    reproducir sonido "explosion"

// Efecto de salto
al presionar espacio:
    si jugador.en_suelo:
        jugador saltar 500
        crear efecto polvo jugador.x jugador.y+32
        reproducir sonido "jump"

// Animaciones
jugador.animacion = "idle"
al mover jugador:
    jugador.animacion = "walk"
Sistema de Vida
// Configurar vida
crear variable vida_jugador 100
crear variable vida_maxima 100

// Barra de vida
crear barra_vida 20 50 200 20

// Recibir daño
al tocar jugador enemigo:
    si no jugador.invulnerable:
        restar vida_jugador 25
        jugador.invulnerable = true
        esperar 2 segundos
        jugador.invulnerable = false

// Game over
si vida_jugador <= 0:
    cambiar escena "game_over"

Consejos y Trucos

Mejores Prácticas
  • Usa nombres descriptivos para objetos
  • Organiza tu código en escenas
  • Prueba frecuentemente durante el desarrollo
  • Usa el modo Ask para aprender nuevos conceptos
  • Combina métodos: manual + comandos + IA
Errores Comunes
  • Olvidar aplicar física a objetos que la necesitan
  • No configurar áreas de colisión
  • Usar coordenadas incorrectas
  • No manejar eventos de colisión
  • Crear demasiados objetos sin optimizar
Atajos Útiles
  • Ctrl + / - Comentar línea
  • Tab - Autocompletar comando
  • F5 - Ejecutar/Probar juego
  • Ctrl + Z - Deshacer
  • Ctrl + S - Guardar proyecto
Optimización
  • Usa pools de objetos para balas/enemigos
  • Destruye objetos fuera de pantalla
  • Limita el número de partículas
  • Usa sprites atlas para mejor rendimiento
  • Evita operaciones costosas en cada frame
